# Phase 07 — 온라인 증분 dedup + 엣지 케이스

## 목표

운영 시나리오(새 글이 순서대로 들어옴)를 시뮬레이션한다. 발행 시각 순
온라인 dedup이 돌아가고, 기존 실험에서 드러난 엣지 케이스(오병합·다일
사가 분할)에 대한 대응이 실험으로 결정된다.

## 작업

- [ ] P07-T01 온라인 증분 dedup 구현 — 발행 시각 순 처리, 직전 24h 활성
      이슈의 대표 임베딩과 비교, 매칭 시 편입+갱신시각 갱신 / 아니면 새 이슈
- [ ] P07-T02 전역 배치 vs 온라인 결과 비교 — 같은 셋에서 두 방식의 군집
      차이를 지표로 측정
- [ ] P07-T03 이슈 대표값 실험 — 최신 글 벡터 vs centroid(벡터 평균) vs
      대표 기사 벡터, 멘토 3안 비교
- [ ] P07-T04 회색지대 LLM 검수 실험 — 임계 구간(예: 0.80~0.90)만 LLM에
      "같은 이슈인가" 판정을 맡겨 오병합 2건이 걸러지는지 측정
      (LLM 호출량·비용도 기록)
- [ ] P07-T05 사가 분할 대응 실험 — 윈도우 확장 vs "활성 이슈 갱신 시
      윈도우 연장" 규칙 비교, 앤더슨(3분할)·킨스키(2분할) 케이스로 검증

## 변경 범위

`src/plick_embedding/pipeline/`, `src/plick_embedding/eval/`, `scripts/`,
`tests/`, `docs/PROGRESS.md`, `docs/DECISIONS.md`

## 완료 조건

- [ ] 온라인 dedup이 56건 셋을 발행 시각 순으로 처리해 이슈 목록을 만든다
- [ ] 대표값 3안의 지표 비교 결과가 기록돼 있다
- [ ] 회색지대 LLM 검수 적용 시 알려진 오병합 2건이 분리되고, 그때의
      LLM 호출 수가 기록돼 있다
- [ ] 사가 분할 대응 방식과 그 근거가 `docs/DECISIONS.md`에 기록돼 있다
