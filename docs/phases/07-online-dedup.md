# Phase 07 — 증분 중복 묶기 + 예외 케이스

## 목표

운영 시나리오(새 글이 순서대로 들어옴)를 시뮬레이션한다. 발행 시각 순
증분 중복 묶기가 돌아가고, 기존 실험에서 드러난 예외 케이스(잘못 합침·여러 날 이어지는
이슈가 쪼개지는 사가 분할)에 대한 대응이 실험으로 결정된다.

## 작업

- [ ] P07-T01 증분 중복 묶기 만들기 — 기사를 발행 시각 순으로 하나씩
      처리한다: 최근 24시간 안에 살아 있는 이슈들의 대표 벡터와 비교해서,
      충분히 비슷하면 그 이슈에 넣고 이슈의 갱신 시각을 새로 찍고, 아니면
      새 이슈를 만든다
- [ ] P07-T02 한꺼번에 vs 하나씩 비교 — 전체를 한 번에 묶는 방식(배치)과
      하나씩 처리하는 방식(증분)이 같은 데이터에서 얼마나 다른 결과를
      내는지 점수로 측정한다
- [ ] P07-T03 이슈 대표값 실험 — 이슈를 대표하는 벡터를 무엇으로 할지
      3가지를 비교한다(멘토 제안 3안): 가장 최신 기사의 벡터 vs 소속
      기사들의 평균(centroid) vs 대표 기사 1건의 벡터
- [ ] P07-T04 애매한 구간 LLM 검수 실험 — 유사도가 애매한 구간(예:
      0.80~0.90)에 걸린 기사만 LLM에게 "같은 이슈인가?"를 물어본다.
      알려진 잘못 합침 2건이 걸러지는지, 그리고 LLM 호출 횟수·비용이
      얼마나 드는지 측정한다
- [ ] P07-T05 길게 이어지는 이슈 실험 — 며칠씩 이어지는 이슈가 24시간
      비교 시간 범위 때문에 쪼개지는 문제(사가 분할)에 대한 두 가지 대응을
      비교한다: 비교 시간 범위 자체를 늘리기 vs 이슈에 새 글이 붙을 때마다 수명을
      연장하기. 실제로 쪼개졌던 사례(앤더슨 3분할, 킨스키 2분할)로 검증한다
