# Phase 06 — 결과 종합 · 최적 구성 선정

## 목표

두 트랙의 wiki 기록을 종합해 우리 서비스의 임베딩 구성(모델 · task_type ·
차원 · 임계)을 근거와 함께 확정한다. 이 결정이 Phase 07~08과 plick-ai
이식의 기준이 된다.

## 작업

- [ ] P06-T01 종합 비교표 작성 — 두 트랙의 실험 노트에서 지표를 모아
      (모델 × 구성)별 ARI·pairwise·유지 임계 구간 폭·비용/쿼터 비교표 생성
      (스크립트로 wiki frontmatter 집계)
- [ ] P06-T02 견고성·운영성 평가 — 최고점만이 아니라 임계 유지 구간 폭,
      API 비용·쿼터·레이턴시, 팀 스택 적합성 평가
- [ ] P06-T03 부족한 실험 보강 — 비교표에서 공백이 보이면 해당 조합 추가
      실험 (양 트랙 협의)
- [ ] P06-T04 최종 선정 — 선정 구성과 근거를 wiki 결정 노트 +
      `docs/DECISIONS.md`에 기록, Confluence(마스터 문서 §11 관련) 공유용
      요약 작성

## 변경 범위

`wiki/`, `scripts/`, `docs/PROGRESS.md`, `docs/DECISIONS.md`

## 완료 조건

- [ ] 두 트랙 전 실험이 포함된 종합 비교표가 wiki에 있다
- [ ] 선정 구성이 점수·임계 구간 폭·비용 3개 축의 근거와 함께
      `docs/DECISIONS.md`에 기록돼 있다
- [ ] 탈락 구성에 대해서도 "왜 아닌지" 한 줄씩 기록돼 있다
- [ ] 팀 공유용 요약(Confluence 게시 가능한 형태)이 준비돼 있다

## 개발자 테스트

- [ ] 선정 구성으로 대표 실험 1회 재실행 — 기록된 점수가 재현되는지 확인
