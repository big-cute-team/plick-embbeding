# phases/

프로젝트를 단계(phase)로 쪼갠 작업 명세. 파일명은 `NN-이름.md`
(`01-project-skeleton.md`처럼 0을 채운 두 자리 번호). 한 phase는
한 세션 안에 끝낼 수 있는 크기로 잡는다 — 크면 쪼갠다.

에이전트는 한 번에 정확히 한 phase만 수행하고 정지한다 (골든 루프,
`docs/AGENT_GUIDE.md` 참조).

## phase 파일 템플릿

아래 섹션 제목은 **글자 그대로** 유지할 것 — 워크플로우 스킬들이
이 제목을 기준으로 동작한다.

```markdown
# Phase NN — <이름>

## 목표

<이 phase가 끝나면 무엇이 가능해지는가, 1~3줄>

## 작업

- [ ] PNN-T01 <작업 한 줄>
- [ ] PNN-T02 <작업 한 줄>

## 변경 범위

<수정해도 되는 파일/디렉토리. 이 밖은 제안만 하고 건드리지 않는다.>

## 완료 조건

<객관적으로 점검 가능한 항목만. "잘 동작한다" 금지, "X 하면 Y가 된다" 형식.>

- [ ] <조건 1>
- [ ] <조건 2>

## 개발자 테스트

<phase 완료 후 개발자가 직접 확인할 것. 명령어/URL 포함, 복붙 가능하게.>

- [ ] <확인 항목 1>
```

## 작성 규칙

- **완료 조건**은 에이전트가 자가 점검하는 기준이다. 측정 불가능한 조건은 쓰지 않는다.
- **변경 범위**는 하드 펜스다. 좁을수록 안전하다.
- **개발자 테스트**는 phase 완료 시 PROGRESS.md의 Developer Test 섹션으로 복사된다.
- 작업 ID는 `PNN-TNN` 형식 (예: `P03-T01`) — 진행 기록에서 이 ID로 참조한다.
