# CONVENTIONS — 코드 컨벤션 (단일 정본)

> 이 리포의 코드 스타일 규칙 정본. 도구가 강제한다: **ruff**(포맷+린트, pyproject.toml).
> CLAUDE.md의 "코드 표준"은 이 문서의 요약이며, 충돌 시 **이 문서가 우선**.

## 커밋/브랜치
`docs/AGENT_GUIDE.md` §3(브랜치)·§4(커밋)를 따른다(여기서 중복하지 않음).
요지: `main ← develop ← <type>/KAN-###-<slug>`, 커밋 `<type>: 요약 (KAN-###)`,
한 태스크=한 커밋, **Co-Authored-By 트레일러 금지**, 커밋 자동·푸시는 승인.

## 코드 스타일

- Python 3.12, uv 관리. 린트·포맷은 ruff 하나로 통일 (`ruff check` + `ruff format`).
- ruff: **line-length 100**, 4-space 들여쓰기, 문자열 **double quote**, 개행 `lf`,
  import 순서(isort) 강제. 커밋 전 `uv run ruff format .` + `uv run ruff check .`.
- 네이밍: 모듈·함수·변수 `snake_case`, 클래스 `PascalCase`, 상수 `UPPER_SNAKE`.
- 공개 함수 시그니처에 타입 힌트 필수(`Any` 지양). 설정·실험 파라미터는 dataclass로.
- 주석 언어는 한국어 허용, **식별자는 영어**.
- API 키는 `.env`(`GEMINI_API_KEY`, `OPENAI_API_KEY`)로만 주입. 코드·커밋에
  키를 넣지 않는다.
- 외부 API 호출은 재시도(지수 백오프) + 부분 실패 시 캐시로 재개 가능하게.
- 테스트: pytest, `tests/`에 소스 구조를 미러링. 외부 API는 목/캐시로 대체하고
  네트워크 없는 환경에서도 전체 테스트가 통과해야 한다.

## 실험 재현성 (이 리포의 핵심 규칙)

- 실행 1회 = `results/<타임스탬프>/` 하나. config(모델·task_type·차원·임계·
  비교 시간 범위·입력 데이터)와 result·report를 함께 저장한다.
- 같은 (텍스트 × 모델 × task_type × 차원) 임베딩은 캐시에서 재사용 —
  같은 입력으로 API를 두 번 호출하지 않는다.
- 랜덤성이 있는 단계는 시드를 고정한다.

## 실험 기록 (LLM Wiki)

- **1차 기록은 `wiki/`(Obsidian 보관함)** — 실험 1회 = `wiki/experiments/`에
  노트 1개 (`YYYY-MM-DD_HHMM_요약.md`), report 모듈이 자동 생성한다.
  노트 머리말 항목(model, task_type, dim, threshold, ari 등)를 채워 조회 가능하게.
- 실험에서 얻은 지식(모델 특성, 임계 감각, 예외 케이스)은
  `wiki/models/`·`wiki/concepts/` 노트에 누적하고 `[[링크]]`로 연결한다.
- 노트를 수동으로 고쳐도 되지만 머리말의 조건·점수는 실행 결과와
  다르게 바꾸지 않는다 (재현성).
- 팀 차원 공유가 필요한 실험(구성 선정 등)은 Confluence "실험 기록
  (임베딩·중복처리)" 폴더에 요약 페이지로 올린다 — 제목·템플릿은 실험 기록
  가이드(페이지 14614543)를 따른다.

## 용어 — 쉬운 말로 쓰기 (사람이 읽는 모든 곳)

문서·리포트 출력·주석·대화 등 **사람이 읽는 텍스트에는 전문 용어 대신 쉬운
말**을 쓴다. 불가피한 기술 용어(ARI, F1 등)는 짧은 설명을 괄호로 붙인다.
코드 식별자(변수·함수 이름)는 예외 — 영어 그대로 둔다.

| 쓰지 말 것 | 대신 쓸 말 |
|-----------|-----------|
| 오병합 | 잘못 합침 (서로 다른 사건을 한 묶음으로) |
| 과분할 | 잘못 나뉨 (같은 사건이 여러 묶음으로) |
| (24h) 윈도우 / 롤링 윈도우 / window | 최근 24시간만 비교 / 비교 시간 범위 |
| ARI | "정답과 얼마나 일치하나 (ARI)" 처럼 설명 병기 |
| pairwise 정밀도/재현율 | "맞게 묶은 비율 / 찾아낸 비율" |
| peak | 최고 지점 / 봉우리 |
| 스파이크 / 칼날 스파이크 | 뾰족한 최고점 (임계값이 조금만 바뀌어도 무너짐) |
| 고원 / plateau | 넓고 평평한 안정 구간 |
| 트레이드오프 | 맞바꿈 (한쪽을 얻으면 한쪽을 잃음) |
| 음성 결과 | 효과 없음 |
| 개체명 과합침 | 같은 이름이면 다른 사건도 합침 |
| 상류 (요약) | 요약을 만드는 앞 단계 |
| (요약) 거절 / 플레이스홀더 | 요약 단계에서 걸러져 진짜 요약 대신 "검수 필요" 안내문만 있는 기사 |
| 라우팅 | 넘김 / 이관 |
| 스택(브랜치) | 위에 쌓은 브랜치 |

새 용어가 생기면 이 표에 추가한다. 명령어 옵션 이름(`--window` 등)은
바꾸지 않되, 설명(help)·문서는 쉬운 말로 쓴다. 어려운 말이 떠오르면 먼저
이 표에서 쉬운 대체어를 찾고, 없으면 쉬운 말로 바꿔 쓰고 표에 추가한다.
