# 잘못 합침과 사가 분할

실험 결과를 정답과 비교할 때 나오는 두 가지 대표적인 오류.

## 잘못 합침 (서로 다른 사건을 한 묶음으로)

정답상 다른 이슈인데 한 묶음으로 붙은 경우. 원인은 대개 [[회색지대]] —
같은 팀·같은 맥락의 다른 선수.

- 예(과거): 제임스+라이스, 아스날 기마랑이스+토날리
  ([[2026-06-29_published56_SEMANTIC_0.85_24h]])
- 예(현재 90건): 홀란 월드컵 소식 3건(경험/교체/응원)이 한 묶음
  ([[2026-07-15_articles90_SEMANTIC_0.85_24h]])
- "맞게 묶은 비율(정밀도)"을 깎는다 → [[정답과 얼마나 일치하나 (ARI)]] 참조.

## 잘못 나뉨 (같은 사건이 여러 묶음으로) — 사가 분할이 주범

같은 이슈인데 여러 묶음으로 쪼개진 경우. 대표 원인이 **사가 분할** —
여러 날 이어진 이슈가 [[최근 24시간만 비교]] 규칙에 걸려 조각남.

- 예(과거): 맨시티 앤더슨 이적이 3덩어리, 킨스키 재계약이 2덩어리
- 예(현재 90건): 에데르손 이적 무산 사가가 날짜별 3조각
- "찾아낸 비율(재현율)"을 깎는다.

## 새로 관찰된 유형 (Phase 05 트랙 B, OpenAI)

- **개체명 과합침** — 같은 선수·팀 이름만 공유하면 서로 다른 사건을 뭉친다.
  OpenAI([[text-embedding-3]])에서 특히 심함: 짧은요약 large/768에선 홀란 관련
  4개 이슈(잉글랜드 저지·게히 훈련·잉글랜드 기원·노르웨이 교체)가 한 덩어리.
  **입력을 상세요약으로 늘리면 완화**(4→2건) — 본문이 개체명 너머 사건을 구분.
  ([[2026-07-15_articles_detail90_title_detail_oai-large-d1536_none_0.65_24h]])
- **언어벽** — 같은 사가라도 언어가 다르면(한글 기사들 사이 영어 트윗 1건) 안
  묶여 사가 분할이 생긴다. 예: 틸레망스 맨유 이적에서 영어 트윗만 따로 떨어짐.
  두 입력 구성 모두에서 공통으로 나타남.

## 왜 구분해서 보나

- 잘못 합침은 **모델·임계값·회색지대·개체명** 문제 → LLM 검수/입력 텍스트로 대응.
- 사가 분할은 **비교 범위 규칙·언어** 문제. 단 비교 범위를 늘려도(window 제거)
  ARI가 거의 안 오른다 — 사가 분할이 줄어든 만큼 시간 떨어진 오합침이 늘어
  상쇄(Phase 05 트랙 B window 분석). 그래서 OpenAI-Gemini 격차는 비교 범위가
  아니라 **순수 임베딩 품질**(개체명 과합침) 문제로 판정.

두 오류는 원인도 대응도 달라서, 채점기(`eval/scoring.py`)가 사례를 나눠서
보여준다.
