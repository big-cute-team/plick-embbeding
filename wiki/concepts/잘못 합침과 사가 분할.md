# 잘못 합침과 사가 분할

실험 결과를 정답과 비교할 때 나오는 두 가지 대표적인 오류.

## 잘못 합침 (서로 다른 사건을 한 묶음으로)

정답상 다른 이슈인데 한 묶음으로 붙은 경우. 원인은 대개 [[회색지대]] —
같은 팀·같은 맥락의 다른 선수.

- 예(과거): 제임스+라이스, 아스날 기마랑이스+토날리
  ([[2026-06-29_published56_SEMANTIC_0.85_24h]])
- 예(현재 90건): 홀란 월드컵 소식 3건(경험/교체/응원)이 한 묶음
  ([[2026-07-15_articles90_SEMANTIC_0.85_24h]])
- "맞게 묶은 비율(정밀도)"을 깎는다 → [[정답과 얼마나 일치하나 (ARI)]] 참조.

## 잘못 나뉨 (같은 사건이 여러 묶음으로) — 사가 분할이 주범

같은 이슈인데 여러 묶음으로 쪼개진 경우. 대표 원인이 **사가 분할** —
여러 날 이어진 이슈가 [[최근 24시간만 비교]] 규칙에 걸려 조각남.

- 예(과거): 맨시티 앤더슨 이적이 3덩어리, 킨스키 재계약이 2덩어리
- 예(현재 90건): 에데르손 이적 무산 사가가 날짜별 3조각
- "찾아낸 비율(재현율)"을 깎는다.

## 새로 관찰된 유형 (Phase 05 트랙 B, OpenAI)

- **같은 이름이면 다른 사건도 합침** — 같은 선수·팀 이름만 공유하면 서로 다른
  사건을 한 묶음으로 뭉친다. OpenAI([[text-embedding-3]])에서 특히 심함: 짧은요약
  large/768에선 홀란 관련 4개 이슈(잉글랜드 저지·게히 훈련·잉글랜드 기원·노르웨이
  교체)가 한 덩어리. **입력을 상세요약으로 늘리면 완화**(4→2건) — 본문이 이름
  너머 실제 사건을 구분해 준다.
  ([[2026-07-15_articles_detail90_title_detail_oai-large-d1536_none_0.65_24h]])
- **언어가 달라 안 묶임** — 같은 사가라도 언어가 다르면(한글 기사들 사이 영어
  트윗 1건) 안 묶여 사가가 쪼개진다. 예: 틸레망스 맨유 이적에서 영어 트윗만 따로
  떨어짐. 두 입력 구성 모두에서 공통. (이 트윗은 요약도 "검수 필요" 안내문이라
  내용이 비어 있던 게 겹친 원인 — 요약이 제대로 있으면 합쳐진다.)

## 새로 관찰된 유형 (Phase 05 트랙 A, Gemini 최고 구성 SEMANTIC/768 @0.86)

남은 오류 7건(잘못 합침 1·잘못 나뉨 6)을 유사도·발행 시각까지 뜯어 본 결과,
네 갈래로 나뉜다 ([[2026-07-22_articles90_gemini-d768_SEMANTIC_0.86_24h]]):

- **같은 인물, 다른 사건 (잘못 합침 1건)** — 홀란 "잉글랜드 선전 기원"(7264) vs
  "노르웨이 경기 교체"(7227), 유사도 0.8708로 기준(0.86)을 살짝 넘음. 둘 다 "홀란
  +월드컵" 맥락이라 임베딩만으론 못 가른다. OpenAI(4개 이슈 합침)보다는 훨씬 덜함.
- **사가 분할 (2건)** — 에데르송 맨유 사가(7/9·7/11·7/13에 걸침), 잉글랜드 훈련
  (7/9 불참↔7/11 참여). 기사끼리 유사도는 기준을 넘는데(0.86~0.90) 발행일이 24시간
  넘게 벌어져 [[최근 24시간만 비교]]에 잘림. 단 창을 끄면 0.9287→0.9100으로 오히려
  하락(잘못 합침 1→8건) — 창이 막아주는 게 더 많다.
- **같은 이슈, 다른 발언 내용 (3건)** — 알론소 첼시 합류(7018 "복귀 느낌 좋다"만
  0.84·0.86으로 탈락), 이라올라 리버풀(합류 소감 vs 2년 계약 설명, 0.7966), 스펜스
  (골키퍼 압박 vs 투입, 0.8308). 같은 사건이라도 **발언·행위의 초점이 다르면**
  유사도가 0.79~0.86에 걸려 못 넘는다. 기준값을 내리면 잘못 합침이 늘어 손해 —
  임베딩 너머의 신호(주체+시각 근접, LLM 판단)가 필요한 영역.
- **노이즈 데이터 (1건)** — "검수 필요 EPL 업데이트" 2건(그린우드/트로사르 각각
  다른 내용의 영어 원문). 라벨상 한 노이즈 이슈지만 실제 내용이 달라 0.8461 —
  요약 파이프라인 품질 문제이지 임베딩 실패가 아니다.

## 왜 구분해서 보나

- 잘못 합침은 **모델·임계값·[[회색지대]]·같은 이름** 문제 → LLM 검수·입력 텍스트로 대응.
- 사가 분할은 **비교 시간 범위·언어·요약 품질** 문제. 단 비교 시간 범위를 늘려도
  ("최근 24시간만 비교" 끄기) ARI가 거의 안 오른다 — 사가 분할이 줄어든 만큼 시간이
  떨어진 잘못 합침이 늘어 상쇄(Phase 05 트랙 B 분석). 그래서 OpenAI-Gemini 격차는
  비교 시간 범위가 아니라 **순수 임베딩 품질**(같은 이름이면 합침) 문제로 판정.

두 오류는 원인도 대응도 달라서, 채점기(`eval/scoring.py`)가 사례를 나눠서
보여준다.
